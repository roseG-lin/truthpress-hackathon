## 四、动画场景详细规格

### Scene 1: A 的狂想 (The Build)

#### 动画流程

```
1. DeepSeek 返回 A 的观点
2. SSE 事件: { event: "A_NEW_STEP", data: { id, text } }
3. 触发动画：
   a. 新方块从顶部掉落 (y: -100 → y: 0)
   b. 掉落时有弹跳效果 (spring)
   c. A 的头像跳到新方块上
   d. B 和 C 在底部显示 "等待" 状态
```

#### Framer Motion 代码参考

```typescript
// 方块掉落动画
<motion.div
  initial={{ y: -100, opacity: 0, scale: 0.8 }}
  animate={{ y: 0, opacity: 1, scale: 1 }}
  transition={{
    type: 'spring',
    stiffness: 300,
    damping: 20
  }}
  className="bg-blue-500 rounded-lg"
>
  {block.text}
</motion.div>

// A 移动到新方块
<motion.div
  layoutId={`agent-a-${block.id}`}
  transition={{
    type: 'spring',
    stiffness: 400,
    damping: 25
  }}
/>
```

#### 时序要求

```
观点生成 → 0.5s 掉落 → 0.3s A 跳跃 → 等待下一步
```

---

### Scene 2: B 的审判 (The Check)

#### 动画流程

```
1. B 开始审查 A 的方块
2. SSE 事件: { event: "B_VERIFY", data: { id, verified: boolean } }

【情况 A：验证通过】
- 方块颜色从蓝色渐变到绿色
- B 跳到方块旁边，与 A 并排
- 显示 ✅ 图标

【情况 B：被推翻】
- 方块震动 (x: [-10, 10, -10, 10, 0])
- 方块破碎效果 (scale: 1 → 0, opacity: 1 → 0, rotate: 0 → 15deg)
- A 向下掉落到下一层（利用 AnimatePresence 自动重排）
- B 获得一个红色方块垫脚
```

#### Framer Motion 代码参考

```typescript
// 验证通过 - 颜色变化
<motion.div
  animate={{
    backgroundColor: ['#3b82f6', '#22c55e']
  }}
  transition={{ duration: 0.5 }}
/>

// 被推翻 - 震动消失
<AnimatePresence>
  {block.status !== 'debunked' && (
    <motion.div
      exit={{
        x: [0, -8, 8, -8, 8, 0],
        opacity: [1, 1, 1, 0],
        scale: [1, 1, 0.8, 0],
        rotate: [0, 5, -5, 15]
      }}
      transition={{ duration: 0.6 }}
    />
  )}
</AnimatePresence>

// A 向下掉落 (利用 layout 自动处理)
```

#### 时序要求

```
B 开始审查 → 0.5s 扫描 →
  ├─ 验证通过: 0.3s 变绿 → B 移动到位
  └─ 被推翻: 0.6s 震动破碎 → A 掉落 (0.5s)
```

---

### Scene 3: C 吸收 D (The Merge)

#### 动画流程

```
1. 用户反驳，生成 D 的观点
2. SSE 事件: { event: "D_NEW_STEP", data: { id, text } }

【前置步骤】
- D 的紫色方块在右侧单独堆叠
- 方块数量：3-5 个

【吸收动作】
3. SSE 事件: { event: "C_MERGE", data: { blocks: [] } }

动画步骤：
a. C 的头像放大，产生漩涡效果 (rotate + scale pulse)
b. D 的所有方块整体向左平移 (x: 300 → x: C的位置)
c. 平移过程中方块稍微错开，形成"搬运"感
d. 方块到达 C 顶端后，逐个堆叠
e. C 的颜色从灰色渐变到紫色
f. C 举起旗帜（最终总结）
```

#### Framer Motion 代码参考

```typescript
// D 的方块整体平移
<motion.div
  animate={{
    x: [300, 50],  // 从右侧平移到 C 的位置
    y: [0, -50, 0] // 稍微上下起伏，增加动态感
  }}
  transition={{
    duration: 1.5,
    ease: 'easeInOut',
    staggerChildren: 0.1  // 子元素交错
  }}
/>

// C 的漩涡效果
<motion.div
  animate={{
    scale: [1, 1.2, 1],
    rotate: [0, 360]
  }}
  transition={{ duration: 1 }}
/>

// C 颜色渐变
<motion.div
  animate={{
    backgroundColor: ['#9ca3af', '#8b5cf6']
  }}
  transition={{ duration: 2 }}
/>

// 旗帜效果
<motion.div
  initial={{ y: 0, opacity: 0 }}
  animate={{ y: [-20, -40], opacity: [0, 1] }}
  transition={{ delay: 2, duration: 0.5 }}
>
  🚩 最终结论
</motion.div>
```

#### 时序要求

```
D 方块生成 → 0.5s/个 掉落 →
C 开始吸收 → 1.5s 平移 + 0.5s 堆叠 →
C 渐变 → 2s 颜色过渡 →
旗帜 → 0.5s 举起
```

## 七、事件格式规范

### 7.1 SSE 事件类型

```typescript
interface LadderEvent {
  event: 'A_NEW_STEP'       // A 生成新观点
       | 'B_VERIFY'         // B 验证结果
       | 'B_DESTROY'        // B 推翻观点
       | 'D_NEW_STEP'       // D 生成反驳
       | 'C_MERGE_START'    // C 开始吸收
       | 'C_MERGE_COMPLETE' // C 吸收完成
       | 'DONE'             // 流程结束
  data: {
    id?: string           // 方块 ID
    text?: string         // 方块文字内容
    verified?: boolean    // 验证结果
    blocks?: Block[]      // D 的方块列表
    summary?: string      // 最终总结
  }
  timestamp: number
}
```

### 7.2 事件示例

```javascript
// A 生成观点
{ event: "A_NEW_STEP", data: { id: "1", text: "远程工作提高效率" } }

// B 验证通过
{ event: "B_VERIFY", data: { id: "1", verified: true } }

// B 推翻
{ event: "B_VERIFY", data: { id: "2", verified: false } }

// D 生成反驳
{ event: "D_NEW_STEP", data: { id: "3", text: "但团队协作会受影响" } }

// C 吸收
{ event: "C_MERGE", data: { blocks: [...] } }

// 完成
{ event: "DONE", data: { summary: "综合来看..." } }
```
