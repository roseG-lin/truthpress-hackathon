## 🎯 核心功能

### 游戏隐喻

| 游戏元素 | 现实映射 | 说明 |
|----------|----------|------|
| **方块 (Blocks)** | 论点/主张 | 每个方块代表一个观点 |
| **梯子 (Ladder)** | 论证过程 | 方块堆叠成上升的梯子 |
| **Agent A** | 建设者 | 建造论点方块 |
| **Agent B** | 审核者 | 验证论点真伪 |
| **Agent C** | 综合者 | 融合不同观点 |
| **Agent D** | 挑战者 | 提出反驳论点 |
| **摧毁方块** | 证伪 | 谎言被揭穿，上方的人掉落 |
| **融合方块** | 共情 | 不同观点被整合 |

## 🎬 动画脚本（4 个阶段）

### Phase 1: The Build (Agent A 的回合)

**触发**: Agent A 生成新论点

**动画序列**:
```
1. 新方块从顶部生成 (y: -100, opacity: 0)
2. 方块掉落并撞击梯子 (弹簧效果：stiffness: 280, damping: 18)
3. Agent A 立即跳到新方块顶部
4. Agent B 和 C 在地面播放"等待"动画 (轻微脉冲缩放)
```

**代码示例**:
```typescript
await buildBlock('AI can personalize learning');
// → 方块生成 → Agent A 跳跃 → 完成
```

---

### Phase 2: The Audit (Agent B 的回合)

**触发**: Agent B 审核论点

**动画序列**:
```
1. Agent B 爬到目标方块
2. 显示"放大镜"图标审查
3. 结果分支:
   
   ✅ 如果为 TRUE:
      - 方块闪烁白色 → 变绿色
      - Agent B 跳到下一个
   
   ❌ 如果为 FALSE:
      - 方块变红色
      - 剧烈震动 (x: [-8, 8, -8, 8, 0])
      - 爆炸消失 (scale: 0, opacity: 0)
      - 关键物理：上方所有方块掉落
      - Agent A 随方块掉落，显示😵表情
```

**代码示例**:
```typescript
await auditBlock(1, false); // 推翻索引 1 的方块
// → 方块变红 → 震动 → 爆炸 → A 掉落
```

---

### Phase 3: The Challenge (Agent D 的回合)

**触发**: 用户点击"我不同意"

**动画序列**:
```
1. Agent D 在右侧建造独立梯子
2. 紫色方块依次生成
3. Agent D 爬上自己的梯子
```

**代码示例**:
```typescript
await challengeBlock('Human empathy builds trust');
// → 右侧生成紫色方块 → Agent D 攀爬
```

---

### Phase 4: The Empathy Merge (Agent C 的终极技能)

**触发**: Agent C 决定综合观点

**电影级动画序列**:
```
1. 吸收：Agent C（在地面）发光
2. 传送：整个 opponentStack（D 的梯子）升起
3. 飞行：D 的梯子飞到主梯子顶部
4. 融合：方块连接到主梯子
5. 胜利：Agent C 在顶部创建金色方块
6. 庆典：所有 Agent 传送到金色方块，五彩纸屑爆炸
```

**代码示例**:
```typescript
await mergeStacks('AI augments human teachers, not replaces them');
// → C 发光 → D 的梯子飞行 → 融合 → 金色方块 → 纸屑
```

  const { buildBlock, auditBlock, mergeStacks } = useLadderGame();

  const handleBuild = async () => {
    await buildBlock('My argument here');

  );

## 🔧 技术实现要点

### 1. 自动掉落物理

```tsx
// 使用 Framer Motion 的 layout 属性
// React 从数组移除元素 → DOM 节点删除 → Framer Motion 自动处理掉落
<AnimatePresence mode="popLayout">
  {blocks.map(block => (
    <GameBlock key={block.id} layoutId={block.id} />
  ))}
</AnimatePresence>
```

### 2. Agent 平滑移动

```tsx
// 使用 layoutId 实现共享元素过渡
<motion.div layoutId={`agent-a-avatar`} />
// → 当 Agent A 的 stackIndex 改变时，自动平滑移动到新位置
```

### 3. 延迟动画队列

```tsx
const runSimulation = async () => {
  await buildBlock('Argument 1');
  await new Promise(r => setTimeout(r, 800)); // 等待动画
  await buildBlock('Argument 2');
  // → 按顺序执行，保持节奏
};
```

## 📊 数据结构

### GameState

```typescript
interface GameState {
  ladderStack: Block[];        // 主梯子
  opponentStack: Block[];      // 挑战者梯子
  agentPositions: {
    A: { stackIndex: number; isFalling: boolean; isDizzy: boolean };
    B: { stackIndex: number; isFalling: boolean; isDizzy: boolean };
    C: { stackIndex: number; isFalling: boolean; isDizzy: boolean };
    D: { stackIndex: number; isFalling: boolean; isDizzy: boolean };
  };
  phase: 'BUILDING' | 'AUDITING' | 'CHALLENGE' | 'MERGING' | 'COMPLETE';
  conclusion?: { initial?: string; final?: string };
}
```

### Block

```typescript
interface Block {
  id: string;          // 唯一标识
  text: string;        // 论点内容
  owner: AgentType;    // 所有者
  status: BlockStatus; // 状态
  color: string;       // 颜色类
}
```

## 🐛 故障排除

### Q: 方块不掉落的？
A: 确保使用了 `<AnimatePresence mode="popLayout">` 和 `layoutId`

### Q: Agent 瞬移而不是平滑移动？
A: 检查是否使用了 `layoutId={`agent-${agent}-avatar`}`

### Q: 动画太快/太慢？
A: 调整 `ANIMATION_DELAYS` 常量
