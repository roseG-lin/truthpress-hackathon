# Agent 求真报社 × SecondMe 答辩稿

## 30 秒版

大家好，我们做的产品叫 **Agent 求真报社 × SecondMe 共识引擎**。  
它解决的问题是：**现在很多讨论平台只有观点对撞，没有可信核查**。  
所以我们做了一个多智能体辩论系统：用户输入一个争议观点后，系统会先自动匹配一个相反立场的真实用户；如果当前没有合适真人，就由 DeepSeek 生成一个可信的反方人格。接着，Agent A 为用户辩护，Agent B 为对手辩护，Judge C 负责抽取事实主张、联网搜索证据，并把结构化裁决显示在中间的求真控制台里。  
我们把 SecondMe 的身份能力和 TruthPress 的求真能力融合起来，让“有分歧的讨论”变成“可验证的讨论”。

---

## 1 分钟版

我们的项目核心是把 **社交身份、观点匹配、AI 辩论、事实核查** 四件事串成一个完整产品。

第一步，用户在辩论咖啡馆输入一个有争议的观点。  
第二步，系统会进入自动对手匹配流程：我们会先在本地观点池里寻找概念上相反的历史观点。如果找到，就让用户和真实立场发生碰撞；如果找不到，就调用 DeepSeek 生成一个“像真实网友一样”的合成对手人格。  
第三步，进入 Arena。这里不是普通聊天，而是一个线性多智能体流程：
- Agent A 替用户立场辩护
- Agent B 代入对手人格进行反驳
- Judge C 负责抽取双方的事实主张、联网搜索证据，并输出结构化 verdict

这个 verdict 会实时进入中间的 **Truth Console**。  
它不仅会告诉你谁更占上风，还会拆解每一条 claim 的证据、裁决结果，以及证据强弱。  
所以我们不是只让 AI 吵架，而是让 AI **在有证据约束的前提下辩论**。

## 3 分钟版

### 一、我们为什么做这个产品

今天很多在线讨论有两个问题：

1. **观点很多，但对手不一定真实存在**
2. **讨论很激烈，但没人验证事实**

所以我们提出一个问题：  
如果能让一个用户自动匹配到“真正反对自己的人”，并让整个交锋过程有一个中立裁判实时验真，会不会更接近高质量共识？

这就是我们做 **Agent 求真报社 × SecondMe 共识引擎** 的原因。

### 二、产品是怎么工作的

产品分成两个阶段：

#### 1. Lobby

用户输入自己的争议观点，比如：
“AI 应该取代教师。”

点击“寻找对手”后，系统不是马上开始和 AI 对话，而是先经过 **Automatic Opponent Matching**。

#### 2. Matchmaker

我们会把用户观点写入一个本地观点池。  
然后优先去找已有的相反立场。  
如果找不到，就让 DeepSeek 生成一个 synthetic opponent persona，包含：
- 名字
- 人设简介
- 开场立场

这一步非常关键，因为它让讨论从“用户对 AI”变成“用户对真实对手 / 拟真对手”。

#### 3. Arena

匹配成功后，系统进入辩论擂台。  
这里采用的是线性多智能体结构，而不是开放聊天：

- **Agent A**：站在用户一方，整理并强化原始观点
- **Agent B**：完全代入对手人格，进行反驳
- **Judge C**：从双方发言中抽取 claims，联网搜索证据，然后输出结构化 verdict

#### 4. Truth Console

这是我们最有辨识度的部分。  
Judge C 不只是说一句“我觉得你对”或者“我觉得你错”。  
它会输出：

- 总体 verdict
- 当前占优方
- 每条 claim 的核查结果
- 支持证据
- 证据强弱

这让观众能一眼看到：  
**谁说得更有道理，不是靠气势，而是靠证据。**

### 三、为什么要和 SecondMe 融合

SecondMe 给我们带来的不是单纯的登录按钮，而是 **身份与社交语境**。

登录用户可以保留自己的 SecondMe 身份和画像；  
匿名用户也可以降级进入，不会阻塞演示。  
这意味着产品既能支持真实用户生态，也适合黑客松现场快速体验。

### 四、我们的亮点

1. **不是单人问答，而是自动对手匹配**
2. **不是普通 AI 辩论，而是带 Judge 的求真辩论**
3. **不是只给结论，而是给结构化证据链**
4. **不是只适合登录用户，也支持匿名现场体验**

### 五、总结

一句话总结我们的产品：

> 我们把“有立场的社交讨论”升级成了“有证据约束的共识生成过程”。

SecondMe 负责“人”，TruthPress 负责“真”。  
谢谢大家。
